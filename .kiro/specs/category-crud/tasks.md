# Implementation Plan

- [x] 1. 基盤設定（ルーティング・認可ポリシー）
- [x] 1.1 (P) カテゴリ管理のルーティングを追加する
  - `config/routes.rb` の admin ネームスペースブロック内に `resources :categories` を追加する
  - `admin_categories_path`・`new_admin_category_path`・`edit_admin_category_path` などのルートヘルパーが利用可能になることを確認する
  - _Requirements: 5.1, 5.2, 5.3, 6.1_

- [x] 1.2 (P) カテゴリ操作の認可ポリシーを実装する
  - 管理者のみが全アクション（index/new/create/edit/update/destroy）を許可するポリシーを作成する
  - 未認証ユーザーは `ApplicationPolicy` の基底実装により自動的にアクセス拒否されることを確認する
  - _Requirements: 5.1, 5.2, 5.3_

- [x] 2. カテゴリ操作ロジックを実装する
- [x] 2.1 (P) カテゴリの作成・更新・削除サービスを実装する
  - 名前を指定してカテゴリを新規作成し、バリデーション結果（成功・失敗）をハッシュで返す
  - 既存カテゴリの名前を更新し、バリデーション結果をハッシュで返す
  - 備品が紐づいていないカテゴリを削除し、紐づきがある場合は削除拒否のエラーメッセージをハッシュで返す（例外を再 raise しない）
  - 戻り値は `{ success:, category:, error:, message: }` の規約に従い、既存サービスと統一する
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 3.1, 3.2, 3.3, 3.4, 4.1, 4.2_

- [x] 2.2 (P) SearchService にカテゴリ検索・ソート機能を追加する
  - カテゴリ名の部分一致検索（大文字小文字を区別しない）に対応する `search_categories` メソッドを追加する
  - 登録日（新しい順・古い順）・カテゴリ名（昇順・降順）・備品数（多い順・少ない順）の 6 種類のソートキーを定数で定義する
  - SQL の LEFT JOIN + GROUP BY + COUNT を使って備品数を集計し、N+1 を回避しつつソートと表示に対応する
  - ソートキーはホワイトリスト定数のみ使用し、SQL インジェクションを防止する
  - 既存の共通 `paginate` メソッドを再利用してページネーションに対応する
  - デフォルトのソート順は「登録日（新しい順）」とする
  - _Requirements: 1.1, 1.2, 1.6, 1.7, 1.8, 1.9, 1.10, 1.11, 1.12, 1.13, 1.14_

- [x] 3. カテゴリ管理コントローラーを実装する
  - `Admin::CategoriesController` を作成し、全 CRUD アクション（index/new/create/edit/update/destroy）を実装する
  - 全アクションで Pundit の `authorize` を呼び出して認可チェックを行う
  - `index` アクションはキーワード・ソート・ページパラメータを SearchService に渡し、一覧と pagination 情報をビューに渡す
  - `create` / `update` / `destroy` は CategoryService に委譲し、成功時はフラッシュ付きリダイレクト、失敗時はフォーム再描画またはエラーフラッシュ付きリダイレクトを行う
  - カテゴリ名のみを許可する Strong Parameters を定義する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 2.1, 3.1, 3.4, 4.1, 4.2, 4.3, 5.1, 5.2, 5.3_

- [x] 4. カテゴリ管理ビューを実装する
- [x] 4.1 (P) カテゴリ一覧画面を実装する
  - キーワード入力フィールド・ソートドロップダウン（6択）・「検索」「リセット」ボタンを含む検索フォームを配置する
  - ソートドロップダウンのデフォルト選択は「登録日（新しい順）」とし、フォーム再送信時に現在の選択値を保持する
  - カテゴリ名・登録日時・備品数のカラムと「編集」「削除」ボタンを含む一覧テーブルを表示する
  - カテゴリが0件のとき「カテゴリが登録されていません」、検索結果が0件のとき「該当するカテゴリが見つかりません」のメッセージを表示する
  - 削除ボタンは `turbo_confirm` による確認ダイアログ付きで実装する
  - ページネーションリンク（前へ・次へ・件数表示）を備品一覧画面と同じスタイルで表示する
  - ページ上部に「＋ カテゴリを追加」ボタンを配置する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 1.6, 1.7, 1.8, 1.9, 1.10, 1.11, 1.12, 1.13, 1.14, 4.3_

- [x] 4.2 (P) カテゴリ作成・編集フォームを実装する
  - 新規作成・編集で共有するフォームパーシャルを作成し、カテゴリ名の入力フィールドとバリデーションエラー一覧を含める
  - 編集フォームは現在のカテゴリ名を初期値として表示する
  - フォーム送信成功後はカテゴリ一覧画面へリダイレクトし、適切なフラッシュメッセージを表示する
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 3.1, 3.2, 3.3, 3.4_

- [x] 5. ナビゲーションに「カテゴリ管理」リンクを追加する
  - グローバルレイアウトの管理者専用ナビゲーションブロック内で「ユーザー管理」リンクの直後に「カテゴリ管理」リンクを追加する
  - カテゴリ管理画面を表示中はリンクをアクティブ状態（青ハイライト）で表示し、他画面では非アクティブスタイルとする
  - 一般ユーザーのナビゲーションには表示しない（管理者専用ブロック内に配置することで制御する）
